Comparativa de Algoritmos de Ordenamiento

Incluye complejidades, tiempos aproximados y recomendaciones de uso.

Made for: ALEHACK (Docente, Alexander Santamaria))
Algoritmo Complejidad (mejor / promedio / peor) Tiempo approx. (ejemplo) Listas recomendadas Tipo de datos y uso real
Bubble Sort
(Intercambio simple)
 O(n) / O(n²) / O(n²) — Cortas (≤ 20) Didáctico, arrays pequeños; funciona con cualquier tipo comparable
Insertion Sort
(Insertar en sublista)
 O(n) / O(n²) / O(n²) — Cortas / casi ordenadas Ideal para listas parcialmente ordenadas; estable; útil en arreglos pequeños y como subrutina
Timsort (ordenamiento por función)
(Python: sorted()/list.sort())
 O(n) / O(n log n) / O(n log n) — Corta, mediana y grande Excelente para casi todo; estable; muy optimizado en Python; usa runs naturales
Quicksort
(Divide y vencerás)
 O(n log n) / O(n log n) / O(n²) — Medianas y grandes Muy rápido en la práctica; inestable por defecto; sensible al pivote
Mergesort
(División y mezcla)
 O(n log n) / O(n log n) / O(n log n) — Grandes Estable; requiere memoria adicional; buen rendimiento consistente
